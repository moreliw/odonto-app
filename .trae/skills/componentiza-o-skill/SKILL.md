# component-architecture

## Objective
Criar componentes reutilizáveis e desacoplados.

## Required Components

- Sidebar
- Header
- KpiCard
- ChartCard
- DataTable
- FilterBar
- Button
- Badge
- Avatar

## Rules

- Cada componente deve ter responsabilidade única
- Evitar lógica misturada com UI
- Props bem definidas
- Tipagem forte (TypeScript)

## Folder Structure

/components
/pages
/services
/hooks ou store

## Behavior
- Componentes reutilizáveis
- Fácil manutenção

## Output
- Código limpo e modular
